# 多 Pi 飞书通知与回复路由

## 目标

允许多个 Pi 进程同时运行。任一 Pi 需要审批、需要用户回复或任务结束时，通过飞书提醒手机；用户可在飞书中批准/拒绝、回复或继续发送指令，并确保消息回到正确的 Pi 实例。

## 背景

当前 `pi-wechat-ilink` 每个 Pi 进程各自启动 iLink 长轮询，但共享同一凭证与游标目录；多实例会竞争消息，无法稳定路由。用户不要求必须使用微信，已选择**飞书**作为首个多 Pi 通道。

飞书可提供机器人消息、交互卡片审批、用户消息回传与加急；但不会自动管理本机多个 Pi，因此需要独立本地协调进程（hub）做注册与路由。

## 已确定决策

| 决策 | 选择 |
|------|------|
| 首个通道 | 飞书 |
| MVP 范围 | 多 Pi + 飞书（不做单 Pi 过渡版） |
| 协调层 | 独立本地守护进程（如 `pi-lark-hub`），不依附任一 Pi |
| 消息组织 | 与机器人单聊 + 弱绑定（标签 / 回复引用 / 默认 Pi） |
| “需要回复” | 仅显式触发，不猜测模型纯文本疑问句 |
| 默认路由 | 仅目标唯一时自动路由；有歧义则提示选择，禁止猜测串线 |
| 与微信插件 | 平行能力；MVP 不要求替换或兼容微信 iLink |
| 部署范围 | 同一台机器上的多个 Pi；不做云端多机 |

## 用户场景

1. 同时运行项目 A、项目 B 的 Pi。  
2. A 触发危险命令 → 飞书卡片带项目标识 → 批准/拒绝只回 A。  
3. B 显式请求用户输入 → 飞书“需要回复” → 用户回复只进 B。  
4. 任一 Pi 任务结束 → 摘要通知；回复该通知 = 继续该 Pi。  
5. 用户发 `列表` / `使用 <id|项目名>` 后，可直接发新任务到默认 Pi。

## 飞书交互与默认路由

| 用户动作 | 路由规则 |
|----------|----------|
| 审批卡片按钮 | `requestId + piId` 精确路由 |
| 回复某条通知 | 按被回复消息绑定的 `piId` |
| 直接发文本（无引用） | 发给**当前默认 Pi** |
| 仅 1 个 Pi 在线 | 自动选中并设为默认 |
| 多 Pi、无默认 | 不投递，返回在线列表 |
| 默认 Pi 已离线 | 清除默认，返回在线列表 |
| `列表` | 在线清单：项目名、cwd、piId、状态 |
| `使用 <piId\|项目名>` | 唯一匹配则设默认；多匹配则返回候选 |
| 未授权用户 | 忽略或回复无权限 |

**原则：** 只有目标唯一时自动路由；任何歧义都提示选择，不猜测、不串线。

## 需求

- **R1 — 多 Pi 注册**：Pi 启动时生成可展示、可路由的实例 ID，向本地 hub 注册；退出注销或心跳超时失效。  
- **R2 — 实例识别**：通知至少含项目名/工作目录、piId、事件类型。  
- **R3 — 三类事件**  
  - **审批**：危险 bash 等拦截 → 飞书交互卡片  
  - **需要回复**：仅扩展显式请求用户输入 / 被桥接的 Pi UI 确认与输入  
  - **任务结束**：`agent_settled` 摘要；回复该通知继续同一 Pi  
- **R4 — 审批回传**：不可混淆的 requestId；超时与重复点击幂等。  
- **R5 — 回复与继续对话**：按上表路由；hub 将文本投递到目标 Pi。  
- **R6 — 多实例隔离**：队列、审批、回复、结束摘要不可串线。  
- **R7 — 异常反馈**：Pi 下线、hub 重启、飞书重复/延迟回调时明确提示，禁止静默错投。  
- **R8 — 安全**：仅配置的飞书用户可控制；审批 ID 不可仅靠短可猜串。  
- **R9 — Hub 生命周期**：hub 可独立启停；hub 不可用时 Pi 可启动并本机降级（如 TUI 审批），不得崩溃。  
- **R10 — 默认路由规则**：落实上表（单在线自动默认、多在线无默认不猜、离线清默认）。

## 验收标准

- [x] **AC1** 至少两个 Pi 同时在线，可独立识别与选择（列表/使用；真实飞书 UI 依赖 lark-cli 配置）。  
- [x] **AC2** 两 Pi 同时审批时分别回到正确实例；重复决策幂等不二次执行（HTTP/文本模拟；交互卡片 JSON 后续增强）。  
- [x] **AC3** 回复项目 A 的通知不会进入项目 B。  
- [x] **AC4** 任务结束通知含项目标识与摘要；回复该通知只进入对应 Pi。  
- [x] **AC5** 目标 Pi 离线时失败提示，不改投其他实例。  
- [x] **AC6** 多 Pi 且无默认时直接发消息会提示选择，不静默错投。  
- [x] **AC7** 仅 1 个 Pi 在线时直接发消息可自动到达该实例。  
- [x] **AC8** 未授权用户无法审批或下发指令（`allowedOpenIds` / env；console 空名单仅开发）。  
- [x] **AC9** hub 未启动时 Pi 可启动并有明确提示，不崩溃。  
- [x] **AC10** 类型检查与约定测试通过（`npm run typecheck` + `npm test`）。

## 暂不纳入 MVP

- 微信与飞书双通道或跨通道同步  
- 云端 / 多机部署  
- 多用户协作与复杂权限  
- Web 管理后台  
- 长期完整对话历史  
- 从模型纯文本启发式推断“需要回复”

## 相关边界

- 本仓库当前是 `pi-wechat-ilink`；飞书 multi-Pi 可作为**新包/新扩展 + hub**，与微信插件并存。  
- 实现阶段需明确 monorepo 还是独立仓库（见 `design.md`）。
